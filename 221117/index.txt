React
 - React란 Facebook에서 만든 라이브러리이다
 - View만을 위해 제작되었다. < npm / yarn을 이용하지만 BackEnd는 아니다
 - yarn이라는 패키지 매니저를 사용한다
 - npm으로 안된다는 이야기는 아니다
 - JavaScript와 HTML 문법을 함께 js 파일에 작성한다 < FM으로는 JSX 파일에 작성을 해야한다
 (*.jsx) (tsx, ts는 정확하게 지켜져야 한다. 단, js, jsx는 크게 따지지 않는다)
  - JSX < JavaScript XML의 약자로 JavaScript를 확장한 문법이라고 쉽게 이야기 할 수 있다
  - React의 대표적인 특징 중 하나
- Component는 구조의 단위. HTML, JavaScript를 포함 한다
- React 상에서만 사용하는 DOM이 있다. 랜더링에 바로 적용되지 않는다
 - 해당 DOM을 Virtual DOM(가상 돔)이라고 한다.
 - React는 Virtual DOM을 수정하고 수정된 부분만을 실제 DOM에 적용 한다
 - let, var 등등은 변화하더라도 React에서 감지 하지 못한다.
 - 변화를 감지 했을 때만 그 부분만을 리랜더링(다시 그리기) 하기 떄문에 let을 사용하는 경우 변화가 없다
 - state라는 단위 ? 개념 ? 것이 있다 < state(뜻 : 상태) 현재 React의 상태를 확인하고 변경 되었을 때
  - 그 변경사항을 React에 알려준다. < 변화 시 리랜더링이 발생한다
 
npm i -g create-react-app
-D < 개발 시에만 사용하는 디펜던시
-g < global : 전역에서 사용하는 라이브러리가 된다

npm list -g < 전역에 설치된 라이브러리를 확인한다.

npx create-react-app testing
 - testing 이라는 폴더를 만들고 그 안에 react를 기본 세팅한다.
