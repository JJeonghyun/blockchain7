API : Application Server << 데이터를 주고 받는 서버
데이터는 보통 json형식으로 주고 받는다.
또는 XML 형식(궁금하면 500원)
REST API
REST API란 Web 통신에서 클라이언트와 서버가 데이터를 주고 받는 가장 대표적인 방식이다
Representational State Trasnfer
GET : 데이터를 받을 때 사용, 단순 데이터 요청, 파일 등등 / 쿼리스트링 : 주소에 출력되는 문자열
POST : 데이터를 보낼 때 사용, 저장할 때 로그인 할때, 회원가입 할때, 비밀스러운 정보를 요청할 때 / body : 숨겨져서 서버에 데이터를 보낸다
PUT : 데이터를 수정할 때 사용, 비밀번호 변경, 게시글 수정
DELETE : 데이터를 삭제할 때 사용
OPTIONS : 요청을 하기 전에 통신 옵션을 설명하기 위해 사용, 로그인을 했다라고 하면 다른페이지가 출력되는데 그 때 로그인 정보를 확인하고 그 이후에 GET을 처리하여 알맞은 데이터(html 파일 등)를 응답한다
- CORS : 크로스 오리진(Cross-Origin-Resource Sharing), 서버와 클라이언트가 하나의 서버에서 html 파일 등등을 갖고 다른 서버에서 미디어(그림, 영상 등등)를 받을 때 두 주소가 다르면 원칙적으로 받을 수 없게 되어 있다. 즉, 불펌이 불가능 하다. 허용할 수 있게 해주는 기술이 CORS이다. 위에서 말하는 주소는 port를 포함 한다.
PATCH : 데이터를 일부만 수정할 때 사용한다.
