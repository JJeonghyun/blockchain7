p : 단락, paragraph 단어의 약자, 블록화되어 있다

br : br은 줄바꿈, 엔터

span : p태그 안에 적기도한다. 거의 유사하고, 인라인 속성을 가지고 있다. 블록화와 대비된다고 볼 수 있다

pre : p나 span태그와 같다. 결정적인 차이가 작성한 그대로 출력된다.

h1,h2,h3,h4,h5,h6 : h1이 제일 큰 제목이다. 이후 숫자가 늘어나면서 점차 작아지는 제목을 생성할 수 있다. (~6)

img : image의 약자로 사용하는 태그. src=이미지의 주소, 파일명 (source), alt = img를 정상적으로 웹페이지상에 출력하지못하면
	해당 자리에 선언한 코멘트를 경고? 이름? 등을 출력해준다.
	./ << 현재 위치 주소를 뜻
	이미지  포맷 : jpg, png, bmp 등
	base64 << 텍스트로 이미지를 볼 수 있다

strong : 굵은 글씨로 강조할 때 사용하는 태그이다. 긴급한 내용이나 알람. 채팅을 쳤는데 공지를 올렸는데 이때 강조하기위해 사용

b : 그냥 굵은 글씨를 나타내는 태그이다. 

del : delete 삭제한 내용. 게시판에서 글을 삭제 했을 때 그 글을 관리자가 확인이 가능해야한다.

ins : insert 입력한 내용. 늘어난 내용을 게시판에 작성했다. 신규 작성한 글, 수정한 글

sup : 윗첨자를 출력할 때 사용한다. 흔히 제곱수

sub : 아래첨자를 출력 할 때 사용한다. 

i : 이름이나 제목 가져온 말들, 인용한 말을 사용하여 기울어진 텍스트를 출력할 때 사용 하는 태그이다.

em : 강조 하기위한 글씨 기울림 태그

style 속성은 해당 태그의 크기, 외곽선, 위치, 배경 등을 설정한다. 기초적인 방법으로 웹페이지 처음엔 html 파일을 주고 받았지만
html 파일이 너무 커졌다. 이에 보는 가독성이 떨어지므로 CSS 파일 (Cascading Style Sheets)로 HTML 파일 외부로 뺌


width 단위 (px, %, em, rem, vw, vh)

px : 픽셀, 한칸한칸을 의미한다.

% : 부모의 기준으로 설정하는 단위. ex) 100% = 부모의 너비

em : 글씨 크기를 기준으로 한다. 부모의 글씨 크기

rem : 글씨 크기를 기준으로 한다. html의 글씨 크기, 모바일, PC를 한번에 만들 때 많이 사용한다.

ex) 글씨 크기가 16px 그럼 1em, 1rem은 16px
글씨 크기가 32px 1em, 1rem = 32px 0.5rem = 16px

vw, vh : viewport width, viewport height 의 약자로 브라우저의 크기나 스마트폰의 크기를 기준. 100vw = 화면의 크기, 100vh = 화면의 높이

width : 영역의 너비
max-width : 영역의 최대 너비
min-width : 영역의 최소 너비
%, vw, vh들은 기준이 있고 그 기준의 비율로 너비를 설정
어떤 내용에 최소 100px 설정이 필요한다면 ? min-width 사용
어떤 내용이 너무 크면 안되고, 1000px를 넘지 않아야 한다면 ? max-width 사용

height : 영역의 높이
max-height : 영역의 최대 높이
min-height : 영역의 최소 높이

width 설정 값이 없다 ? 부모의 너비를 받아오는데 사용자가 margin 설정값을 10px로 설정했으니
margin(외부영역)을 우선적으로 만들어줘야하기에 부모의 너비가 아닌 설정값으로 적용 된다.

너비는 부모의 너비를 따라 받는다. 높이는 자식의 높이를 따라 간다. 즉, 설정을 하지 않으면 0px로 설정되기에 조심해야한다.
